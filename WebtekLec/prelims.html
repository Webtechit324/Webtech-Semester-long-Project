Internet

  • A global system of interconnected computer networks that use the standard Internet Protocol Suite (TCP/IP) to serce billions of users worldwide.
  • It is a networks that consists of millions of private, public, academic, business and government networks, of local to global scope, that are linked by a broad array of electronic, wireless and optical networking technologies.

Vinton Cerf

  • Father of the Internet
  • He is a co-designer of the TCP/IP and the archictecture of the Internet.

History of Internet

    October 24 1995
  • Internet means a global information system that is logically linked together by a globally unique address space based on the internet protocol (IP).
  • Transmission Control Protocol (TCP) means sending information over the internet and this provides high level services publicly or privately. This also is a platform that create higher level services for users.
    1969 (established)
  • Four computers (that talks to one another) connected to the Advance Research Project Agency Network (ARPANET) by Defense Advanced Research Projects Agency (DARPA).
    1972
  • Electronic mail (e-mail) - the first killer application was deploy on the web.

World Wide Web (WWW 1989)

  • A global information medium which users can read and write via computers connected to the Internet. The term is often mistakenly used as a synonym for the Internet itself, but the Web is a service that operates over the Internet, as electronic mail (e-mail) does.

Tim Berners-Lee

  • Father of the Web
  • In September 1994, Tim Berners-Lee founded the World Wide Web Consortium (W3C) at the Massachusetts Institute of Technology with support from the Defense Advanced Research Projects Agency (DARPA) and the European Commision. It comprised various comapanies that were willing to create standards and recommendatiopns to improve the quality of the Web.
  • Made the Hypertext Transfer Protocol (HTTP), Hypertext Markup Language (HTML), Unified Resource Locator (URL), Web Server, Web CLient and Hyperlinking of Documents

Wide Area Information Servers (WAIS)

  • periodically queries databases and get the result and prints what you needed.

Hyper Text Transfer Protocol (HTTP) hypermedia

  • Application layer used primarily to retrieve hypertext (on hypermedia) documents and reources on the World Wide Web.
  • jointly developed by the W3C and the IETF.
    Version History
  • HTTP 0.9 (1991)
  • Phases: Connection -> Request -> Response -> Disconnection (4 steps)
  • HTTP 1.0 (Request For Comments (RFC) 1945, May 1996)
  • HTTP 1.1 (RFC 2068, Jan 1997) (RFC 2616, Jun 1999) (RFC 7230-7235, Jun 2014)
  • HTTP 2.0 (RFC 7540, May 2015)

Protocol

  • A set of rules that needs to be followed when messages are communicated in the Internet.
  • Examples:

    • HTTP(Hypertext Transfer Protcol) - a protocol used for tranferring messages over the Internet.

    HTTPs - a protcol with a combination of HTTP and SSL or Secure Sockets Layer which makes the sharing of information more secure.

HTTP Response Message

  • Status Line

    • The first line of the response message. It start with the protocol version and the numeric status code with the description.

  • Response Headers

    • This allows the server to pass additional messages that cannot be placed in the Status Line.

  • Message Body

    • The message body is used to carry the entity-body associated with the request or response. It only differs from the entity-body when transfer coding has been applied.

HTTP Request Methods

  • GET

    • Request data from a specified source.

  • HEAD

    • Same as get but returns only HTTP headers and no document body.

  • POST

    • Submit data to be processed to a specified resources.

  • OPTIONS

    • OPTIONS method represents a request for information about the communication options available on the request/response chain identified by the Request-URI.

  • TRACE

    • TRACE method is used to invoke a remote, application-layer loop- back of the request message..

  • PUT

    • Uploads a representation of the specified URI.

  • DELETE

    • DELETE method requests that the origin server delete the resource identified by the Request-URI.

  • CONNECT

    • Converts the request connection to a transparent TCP/IP tunnel

HTTP Message Headers

  • General Header Fields

    • This header fields have general applicability for both request and response messages.

    - connection

    - data

    - pragma

    - transfer encoding

    - trailer

    - upgrade

    - via

    - warning

  • Request Header Fields

    • The request-header fields allow the client to pass additional information about the request, and about the client itself, to the server. These fields act as request modifiers.

    - accept language

    - accept encoding

    - authorization

    - proxy

    - expect

    - from

    - host

  • Response Header Fields

    • The response-header fields allow the server to pass additional information about the response which cannot be placed in the Status- Line. These header fields give information about the server and about further access to the resource identified by the Request-URI.

    - accept - range

    - age

    - ETag

    - Location

    - Proxy - authenticate

    - WWW - authenticate

    - retry - after

    - server

    - vary

  • Entitty Header Fields

    • This header fields define meta information about the entity-body or, if no body is present, about the resource identified by the request.

    - allow

    - content - encoding

    - content - language

    - content - length

    - content - location

    - content - MDS

    - content - range

    - content - type

    - expires

    - last - modified
